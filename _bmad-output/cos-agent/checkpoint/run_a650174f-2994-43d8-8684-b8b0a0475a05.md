## Current Delivery State

### Infrastructure (Epic 1)

All four infrastructure stories are **code-complete** with clean TypeScript compilation and passing production builds. The following components have been delivered:

**Server side:**
- Express 5 app with CORS (localhost:5173), JSON body parser, graceful shutdown (SIGTERM/SIGINT)
- SQLite database at `~/.agent-studio/db.sqlite`: WAL mode, foreign keys, idempotent migration runner (`PRAGMA user_version`), 5 tables (projects, executions, messages, logs, approvals)
- `GET /api/health` endpoint returning `{ok, dbPath, version}`
- WebSocket server (`packages/server/src/websocket/WebSocketServer.ts`) attached to HTTP server; per-executionId subscription tracking; `broadcast(executionId, msg)` and `broadcastAll(msg)` exports; parses `subscribe`/`unsubscribe` client messages

**Client side:**
- React 18 + BrowserRouter + QueryClientProvider entry point (`main.tsx`)
- 7 routes wired (onboarding, projects, chat, history, execution detail, settings, 404)
- Placeholder views for all 7 routes with dark `bg-gray-950` theme and chat-first inbox structural hints
- Zustand store skeleton (`appStore.ts`) with all typed fields: `projects`, `selectedProjectId`, `activeExecutions`, `currentExecutionId`, `wsConnected`, `reconnectAttempts`, `isOnboarded`, `settings`, `logPanelOpen`
- `api/client.ts` fetch wrapper with `VITE_API_URL` env var support
- `useWebSocket.ts` singleton hook with exponential backoff (1s→2s→4s→8s→16s→30s cap), Zustand `wsConnected`/`reconnectAttempts` sync, `send()` and `useWsMessage()` exports
- Tailwind CSS v3 configured via `tailwind.config.ts` + `postcss.config.js`

## Design Differentiation Summary

**Intended archetype:** chat-first inbox

- Dark `bg-gray-950` background on all views (avoids generic white/gray scaffold)
- Indigo left accent bar (`w-1 bg-indigo-600`) in ProjectInboxView placeholder — establishes inbox silhouette before thread rows exist

**Layout silhouette difference from default:** ProjectInboxView has a narrow vertical accent bar + inbox column (not a centered card grid). ChatView has an explicit 3-column skeleton with left border dividers.

**Avoided default convergence risks:** No centered hero layout; no light-background card dashboard; no default Vite counter boilerplate.

**Unresolved weak spots:** Skeleton placeholder text labels are still visible in all views; full inbox thread row structure (status badges, relative timestamps, hover states) arrives in story-2.3.

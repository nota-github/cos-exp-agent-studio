## Design Differentiation Summary

- **Intended archetype:** chat-first inbox
- **Visible choices made (delivered so far):**
  - Project list uses email-inbox-style thread rows (icon rail + avatar + two-line metadata) — not a centered card grid
  - Indigo left-border accent on hover (`before:` pseudo-element) signals item selection
  - Korean-specific relative timestamps ("3시간 전", "없음") and action copy ("추가", "폴더 탐색", "선택 완료") avoid generic scaffold language
  - AddProjectModal uses dark `gray-900` with `indigo-600` accents; directory browser is an inline accordion, not a blocking dialog
- **Layout silhouette difference from default:** Two-pane layout (narrow icon rail + thread list) is structurally distinct from a centered card dashboard. A user landing on `/projects` would see an email-app-like interface, not a typical CRUD list.
- **Avoided default convergence risks:** No white card grid, no generic "Submit/Cancel" buttons, field errors are inline (not toast-only)
- **Unresolved weak spots:**
  - Inbox currently has no sidebar (planned for story-6.2); the archetype is incomplete until navigation is wired
  - ChatView, OnboardingWizard, SettingsView, history views have no archetype-specific implementation yet — these are high-risk for defaulting to generic scaffold UI when built

## Design Differentiation Summary

**Intended archetype:** chat-first inbox

**Visible choices:**
- Dark `bg-gray-950` base; indigo left accent bar in ProjectInboxView
- ChatView three-column layout (sidebar | chat | log slot) matching messaging app structure
- Thread-row metaphor in ProjectInboxView and ExecutionHistoryView (vs. card grids)
- Hover-reveal rerun actions in MessageBubble (ephemeral, no modal or side panel)
- Approval cards (epic-7, pending) designed to appear as inline action cards within the chat flow

**Layout silhouette difference from default:** Left sidebar + fixed bottom input bar + message timeline center area is structurally distinct from a centered card dashboard. The first screen (ProjectInboxView) uses a narrow left accent bar + inbox list, not a hero/card layout.

**Avoided default convergence risks:** No generic centered card grid on first screen; no plain list without status/timestamp metadata; no generic "Submit"/"Cancel" labels.

**Unresolved weak spots:**
- ApprovalCard visual differentiation from chat bubbles not yet verified (epic-7 not implemented)
- Browser runtime rendering not confirmed; only code-level design compliance assessed

## Outcome Quality Summary

**Assessment: Meaningfully better than direct-agent baseline for the implemented scope.**

A non-developer user can:
1. Complete a 4-step onboarding wizard in Korean with inline validation
2. Register a project via the inbox-style UI without typing a CLI command
3. Submit a natural language task request via the chat input bar and see it execute
4. Monitor execution status in real-time (status badge + log panel)
5. View conversation history with previous requests and agent responses
6. Re-run or edit-and-rerun any previous task from chat hover actions
7. Review execution history with log accordion and re-run from detail view

**Remaining gap vs. baseline:** The approval confirmation flow (epic-7) is absent. Users cannot approve or reject sensitive operations (file modifications, package installs) through the UI — the CLI process will be paused indefinitely at approval prompts without a way to respond. This is the single largest remaining gap between the current product and full MVP acceptance.

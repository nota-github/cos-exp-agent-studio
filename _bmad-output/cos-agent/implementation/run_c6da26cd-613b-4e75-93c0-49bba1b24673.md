## Selected Story

**story-3.1 — CommandBuilder: injection-safe CLI argument builder**

Goal: Implement the security-critical CommandBuilder module that constructs CLI argument arrays for spawn calls, enforcing injection-safe practices as the single source of truth for all CLI invocations.

---

| Criterion | Evidence |

## Deferred Work

- **OQ1 (open):** OpenClaw's exact flag names (`--prompt`, `--project`, etc.) are assumed based on the architecture doc. The `OpenClawAdapter.buildArgs` method uses these flags; they will need to be verified against the real CLI spec before ProcessManager integrates this module (story-3.2).
- **OpenClawAdapter.parseResult:** Minimal implementation — returns last 5 non-empty lines as summary. Proper output parsing deferred until OpenClaw JSON output spec is confirmed (OQ1, story-3.4).
- **SettingsManager integration:** `buildSpawnArgs` accepts `cliPath` as an explicit first parameter. When story-3.3 integrates CommandBuilder, it will read `cliPath` from SettingsManager and pass it in. No coupling to SettingsManager in this slice.
- **Additional CliAdapter implementations:** Only `OpenClawAdapter` implemented. Interface designed for extension.

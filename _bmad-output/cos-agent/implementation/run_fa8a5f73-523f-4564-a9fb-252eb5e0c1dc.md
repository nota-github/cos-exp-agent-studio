## Selected Story

**story-5.5 — App initialization gate — redirect to onboarding if not configured**

Goal: Ensure any user who hasn't completed setup is redirected to the onboarding wizard before accessing any other part of the app, and that execution is blocked with a helpful message if settings become incomplete.

---

## Quality Plan

| Criterion | Description | Implementation Evidence |
|-----------|-------------|------------------------|
| AC1 | First visit (unconfigured): any route redirects to `/onboarding` | `useEffect` in `App.tsx` calls `GET /api/settings/status`; if `isConfigured=false` and path ≠ `/onboarding`, calls `navigate('/onboarding', { replace: true })` |
| AC2 | Subsequent visits (configured): `/` redirects to `/projects` | Existing `<Route path="/" element={<Navigate to="/projects" replace />} />` plus gate sets `isOnboarded=true` |
| AC3 | After onboarding (configured): visiting `/onboarding` redirects to `/projects` | Gate detects `isConfigured=true && initialPath === '/onboarding'` → navigates to `/projects` |
| AC4 | Missing settings shows inline guidance in ChatView with link to `/settings` | `ChatView` state `missingSettings: string[] \| null`; displayed below input area with `Link` to `/settings` |
| patch | OnboardingWizard copy fix: step 4 error message now matches actual button behavior | Updated line 401 copy to `"이전" 버튼으로 이전 단계로 돌아가 설정을 수정할 수 있습니다.` |

## Design Compliance Notes

**Status: pass**

- AC1/AC3 gate keeps users from seeing app content before setup — aligns with "프로젝트 선택 → 요청 입력" flow contract
- Brief loading state (`bg-gray-950` full screen) avoids flash of wrong route without breaking dark theme
- Missing settings warning uses amber color to indicate actionable state, not failure

---

## Design Differentiation Notes

- **Status:** pass
- **Intended archetype:** chat-first inbox
- **Visible choices made:** inline warning in the message input area (not modal/toast) preserves chat-thread metaphor; link-styled CTA ("설정 화면으로 이동") matches thread-action language
- **Layout silhouette difference from default:** no floating error overlay; warning is anchored to the input bar
- **Default convergence risks avoided:** no generic "please configure first" page; redirect is invisible to users who are configured
- **Remaining weak spots:** ChatView input is still minimal (single-row textarea); full message-history UI comes in story-6.2

---

## Outcome Quality Notes

- **Purpose clarity:** Gate is invisible to configured users; unconfigured users land directly in the wizard. First screen explains purpose.
- **Primary action:** For unconfigured users — complete setup. For configured users — send a task.
- **State/copy handling:** Loading state is a blank dark screen (brief). Missing settings uses "실행 전에 … 설정이 필요합니다" — product-specific, not generic.
- **Visible quality choices:** `initialPath` ref prevents stale closure in useEffect; `active` flag prevents state update after unmount; server unreachable is handled gracefully (gate fails open).
- **Remaining weak spots:** No loading indicator during the status check (just blank dark screen).

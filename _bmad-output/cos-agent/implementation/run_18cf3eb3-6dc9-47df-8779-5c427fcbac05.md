## Implementation Summary

### WebSocketServer.ts
- Imported `db` from `../db/index.js`
- Added `PendingApprovalRow` interface
- On every `connection` event: query `SELECT * FROM approvals WHERE status = 'pending'` and immediately send `{type:'connected', pendingApprovals: row[]}` to the new client
- This ensures any client (initial load OR reconnect) receives the current pending approvals list

### Toast.tsx (new)
- Simple fixed-position overlay toast (bottom-center)
- Auto-dismisses after `duration` ms (default 4000ms) via `setTimeout`
- Called with `onDismiss` callback that clears parent toast state
- Dark gray styling consistent with the rest of the dark-mode UI

### ChatView.tsx
- Added `toastMessage` state (`string | null`)
- Added `handleToastDismiss = useCallback(() => setToastMessage(null), [])` ‚Äî stable reference prevents Toast's `useEffect` timer from resetting on each WS-driven re-render
- Imported and rendered `<Toast>` when `toastMessage !== null`, passing `handleToastDismiss` (not an inline lambda)
- Extended the `useWsMessage` callback to handle `type === 'connected'`:
  - Reads current React Query cache via `queryClient.getQueryData`
  - Filters out approvals whose cards already exist (by `ws-approval-${id}` or metadata match)
  - Calls `queryClient.setQueryData` to inject new approval cards
  - Shows toast with count if any were injected
- Dedup strategy: primary check on cached data before `setQueryData`; secondary check inside the updater function for race-condition safety

## Selected Story

## Quality Plan

| Criterion | Code Evidence |
|---|---|
| Messages load on chat view open | `useMessages(projectId)` react-query hook calls `GET /api/projects/:id/messages`; renders in center panel |
| User messages right-aligned | `MessageBubble` type='user' → `flex justify-end` + indigo bubble |
| Agent messages left-aligned | `MessageBubble` type='agent' → `flex justify-start` + gray bubble + avatar icon |

## Design Compliance Notes

---

## Design Differentiation Notes

**Status: pass**
**Intended archetype:** chat-first inbox

**Visible choices made:**
- Three-panel layout: compact sidebar (w-56) + message area + collapsible log panel
- User messages: right-aligned indigo bubble, no label — same convention as iMessage/WhatsApp
- Agent messages: left-aligned gray bubble with circular avatar icon — distinct from user, not just mirrored
- System events: small centered pill (no bubble) — reads as metadata not conversation
- File-change pills: small indigo-tinted monospace tags below agent bubble — secondary to content

**Layout silhouette difference from default:**
- Not a generic card list; three-zone layout with sidebar, scroll-area, and anchored input bar is clearly a messaging app pattern

**Default convergence risks avoided:**
- No generic centered card/list layout
- No plain table/list for messages

**Remaining weak spots:**
- Timestamps not shown per message (deferred — not in story-6.2 scope)
- Sidebar shows path as secondary text; could be truncated badly on very long paths (not a blocking issue)

## Outcome Quality Notes

**Purpose clarity:** On chat view open, user immediately sees the conversation history in a messaging-app layout. Empty state explains what to do next.

**Primary action:** Submit textarea → messages appear; WS event triggers message refresh automatically.

**State/copy handling:**
- Loading: "메시지를 불러오는 중..." centered in message area
- Empty: icon + "아직 대화 내역이 없습니다" + "아래 입력창에 작업을 요청해 보세요"
- Populated: chronological bubble timeline

**Visible quality choices:**
- `invalidateQueries` on both successful submission and `execution_complete` event ensures user and agent messages both appear without manual reload
- `scrollIntoView({ behavior: 'smooth' })` triggered on message list change for auto-scroll
- Sidebar highlights active project with indigo tint + border

**Remaining weak spots:**
- GUIDED_VERIFICATION_REQUIRED: browser runtime confirmation that message bubbles render correctly and auto-scroll works after an execution completes. Start server with `npm run dev`, navigate to a project chat view, submit a task, and confirm: user bubble appears immediately after submission, agent bubble appears after execution completes, scroll reaches the bottom.

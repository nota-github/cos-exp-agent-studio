## Selected Story

**story-7.5 — Approval result in execution history**

Goal: Record approval decisions as messages in the chat flow and display the full approval history in the execution detail view, so users can audit what was approved or rejected during any past execution.

---

| Criterion | Intended Evidence | Location |

| ExecutionDetailView "승인 내역" section shows all approval decisions for that execution | New section renders approval request + result pairs extracted from useExecutionMessages; shows action_type, target, risk badge, decision indicator | ExecutionDetailView.tsx |
| Approval result appears in chat as a brief status pill after the ApprovalCard | `type='approval_result'` branch in MessageBubble renders colored rounded-full pill (green for approved, red for rejected); WS `approval_result` event handler in ChatView appends to message cache in real time | MessageBubble.tsx, ChatView.tsx |
| Empty state shown when execution had no approvals | Section always renders with fallback text "이 실행에서 승인 요청이 없었습니다" when `approvalHistory.length === 0` | ExecutionDetailView.tsx |

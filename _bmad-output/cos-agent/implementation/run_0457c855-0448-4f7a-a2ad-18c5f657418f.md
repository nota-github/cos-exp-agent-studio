## Outcome Quality Notes

- **Purpose clarity:** The messages table now durably records every user request and agent response, enabling the chat history to survive page refreshes and server restarts. This directly enables story-6.2 (ChatView conversation display).
- **Primary action:** User message insert is tightly coupled to execution start — no orphaned messages if spawn fails.
- **State/copy handling:** System messages use Korean copy consistent with the rest of the app (`'CLI 실행에 실패했습니다: ...'`, `'실행이 중단되었습니다'`).
- **Visible quality choices:** `metadata` for agent messages includes `fileChanges` and `commandsRun` only if present (null otherwise), keeping the column clean.
- **Remaining weak spots:** GUIDED_VERIFICATION_REQUIRED — DB row insertion cannot be confirmed without a running server and sqlite3 CLI inspection.

## Selected Story

**story-5.2 — Settings REST API**

Expose REST endpoints for reading and writing settings, and for testing that the CLI tool is operational, so the frontend can manage configuration through the API.

---

## Quality Plan

| Criterion | Implementation Evidence |
|-----------|------------------------|
| AC1: `GET /api/settings` returns all config fields with API key masked | `settingsRouter.get('/')` calls `buildMaskedConfig()` which returns `config` fields + `apiKey: '•••••'` (if set) or `''` + `apiKeySet: boolean` |
| AC2: `PUT /api/settings` with `{apiKey}` stores in keychain; GET shows `apiKeySet: true` and masked value | `settingsRouter.put('/')` zod-validates body, calls `settingsManager.setApiKey(apiKey)` for apiKey field, returns `buildMaskedConfig()` |
| AC3: `POST /api/settings/test` with valid CLI path returns `{ok: true, output: "[version string]"}` | `settingsRouter.post('/test')` uses `spawnSync(cliPath, ['--version'], {shell:false, timeout:5000, encoding:'utf-8'})`; returns `{ok:true, output: stdout.trim()}` when `result.status === 0` |
| AC4: `POST /api/settings/test` with bad path returns `{ok: false, error: 'CLI 도구를 실행할 수 없습니다: [reason]'}` | Error branches in `/test`: `result.error` case, non-zero exit code, and caught exception all return `{ok:false, error:'CLI 도구를 실행할 수 없습니다: ...'}` |
| AC5: `GET /api/settings/status` returns missing fields when settings incomplete | `settingsRouter.get('/status')` delegates to `settingsManager.isConfigured()` and returns `{isConfigured: status.ok, missing: status.missing}` |

   ```bash
   curl -X PUT localhost:3000/api/settings -H 'Content-Type: application/json' -d '{"apiKey":"sk-test"}'
   ```


4. **AC4 — POST /api/settings/test with bad cliPath:**
   ```bash
   curl -X PUT localhost:3000/api/settings -H 'Content-Type: application/json' -d '{"cliPath":"/usr/bin/nonexistent-cli"}'
   ```

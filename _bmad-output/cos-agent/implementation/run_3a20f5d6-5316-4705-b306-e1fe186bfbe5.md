---

## Design Differentiation Notes

**Status: pass**
**Intended archetype:** chat-first inbox

- Visible choices: bottom-anchored input bar with product-specific "작업 실행" CTA mirrors messaging app send pattern; "중단" replaces "작업 실행" in-place to keep the interface uncluttered during execution
- Layout silhouette difference: fixed input bar at bottom with conditional action button is a recognizable chat-inbox pattern distinct from a form/dashboard default
- Default convergence risks avoided: no generic "Submit" label; stop affordance is in the input zone (not a separate control panel)
- Remaining weak spots: textarea is single-line by default (grows on input); no visual indicator for "running" state in the input bar itself beyond the button swap

---

## Outcome Quality Notes

- **Purpose clarity:** The input bar with "작업 실행" copy makes the primary action self-evident for non-CLI users
- **Primary action:** "작업 실행" → optimistic message appears → execution starts; the full flow is now wired end-to-end
- **State/copy handling:** Empty state disables button; running state swaps to red "중단"; missing settings shows amber warning with actionable link
- **Visible quality choices:** Optimistic message gives instant feedback before server round-trip; auto-resize prevents textarea overflow on long inputs
- **Remaining weak spots:** GUIDED_VERIFICATION_REQUIRED items listed above

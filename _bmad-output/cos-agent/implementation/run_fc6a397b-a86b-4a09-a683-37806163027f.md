## Selected Story

**story-7.1: ApprovalService — signal detection and process pause**

Goal: Detect when the CLI tool requests user approval, pause the CLI process, persist the approval request, and broadcast the event to the frontend.

---

## Quality Plan

| Criterion | Code Evidence |
|---|---|
| AC1: JSON approval protocol detected | `parseApprovalSignal()` — JSON path validates `{type:'approval_request', action_type, target, risk_level, description}` |
| AC2: Pattern fallback detected | `parseApprovalSignal()` — iterates `APPROVAL_PATTERNS` array, returns generic `ApprovalRequest` on match |
| AC3: No false positives | `parseApprovalSignal()` returns `null` for non-matching empty/plain lines |
| AC4a: `approvals` row inserted (status=pending) | `handleApproval()` — `INSERT INTO approvals ... status='pending'` |
| AC4b: `messages` row inserted (type=approval_request) | `handleApproval()` — `INSERT INTO messages ... type='approval_request', metadata={approvalId}` |
| AC5: WS broadcast `approval_request` event | `broadcast(executionId, {type:'approval_request', executionId, approval:{...}})` |
| AC6: execution status = `approval_pending` | `UPDATE executions SET status='approval_pending' WHERE id=?` |
| AC7: No stdin while pending | `processManager.pause(executionId)` sets `managed.paused=true`; `write()` returns early when paused |

## In Scope for This Run

- `packages/server/src/services/approvalPatterns.ts` — new file, exported `APPROVAL_PATTERNS: RegExp[]` constant array (9 patterns covering English/Korean, JSON protocol and CLI prompts)
- `packages/server/src/services/ApprovalService.ts` — new file; `ApprovalRequest` interface, `parseApprovalSignal()` function, `ApprovalService` class, `approvalService` singleton

- `packages/server/src/app.ts` — modified: imports `approvalService`, calls `approvalService.setup()` after `setupWebSocket(server)`

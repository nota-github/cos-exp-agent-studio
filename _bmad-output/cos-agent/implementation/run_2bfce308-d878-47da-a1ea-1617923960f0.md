## Design Compliance Notes

## Design Differentiation Notes

**Status: pass**  
**Intended archetype:** chat-first inbox  
**Visible choices made:** Centered max-w-lg focused wizard panel with no sidebar or navigation rail (deliberately stripped from the settings pattern); step labels and counter as the only chrome; large step headings with single-purpose inputs per step — closer to a messaging "setup card" than a settings form.  
**Layout silhouette difference:** The wizard is a single centered card, not the split left-rail + scrollable panel layout used elsewhere. Users see one question at a time, matching a conversational onboarding pattern.  
**Default convergence risks avoided:** No generic multi-section settings form; no navigation rail on the onboarding screen; step counter "1 / 4" makes progress concrete.  
**Remaining weak spots:** The "찾아보기" directory browser expands inline which can make the card feel tall; this is acceptable for a setup wizard context.

## Outcome Quality Notes

**Purpose clarity:** First-time user landing on `/onboarding` sees "CLI 도구 경로 설정" with a single input and "다음" — immediately clear what to do.  
**Primary action:** "다음" is the clear CTA on steps 1–3; "시작하기" on step 4 success is specific and action-completing copy.  
**State/copy handling:** Empty validation (AC2), spinner (AC4), success card (AC5), failure card with fix instructions (AC6) — all states handled with product-specific Korean copy, not placeholder text.  
**Visible quality choices:** Each step has a focused heading + description + single input; no simultaneous multiple fields; "이전" always present on steps 2–4; "건너뛰기" / "나중에 설정" for optional steps convey that the user is in control.  
**Remaining weak spots:** Runtime browser confirmation pending; story-5.5 needed for automatic redirect to onboarding on first launch.

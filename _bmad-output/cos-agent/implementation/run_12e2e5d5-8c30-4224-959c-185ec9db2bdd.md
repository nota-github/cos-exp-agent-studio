## In Scope for This Run

- Full rewrite of `packages/server/src/services/SettingsManager.ts`
  - `Config` interface with camelCase fields (replaces old snake_case `Settings`)
  - `loadConfig()` synchronous — creates `config.json` with defaults if absent
  - `saveConfig(partial)` synchronous — merges and writes
  - `getApiKey(): Promise<string | null>` — keytar with env var fallback
  - `setApiKey(key): Promise<void>` — keytar storage
  - `isConfigured(): Promise<{ok, missing}>` — checks cliPath executable + API key
  - Constructor accepts optional `configDir` for testability
  - Exports class `SettingsManager` and singleton `settingsManager`
- New `packages/server/src/services/__tests__/SettingsManager.test.ts`
  - 5 tests covering AC1, AC2, AC5, AC6 (file operations and path validation)
  - AC3 and AC4 noted as GUIDED_VERIFICATION_REQUIRED
- Update `packages/server/src/routers/executionRouter.ts`
  - POST `/` handler made async (Express 5 handles async natively)
  - `isConfigured()` call awaited; response shape changed from `{configured}` to `{ok}`
  - Replaced `getSettings()` + `settings.api_key` with `loadConfig()` + `await getApiKey()`
